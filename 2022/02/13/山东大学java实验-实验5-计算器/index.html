Java实验——实验5 计算器GUI制作

发表于2022-02-13|更新于2024-05-08|课程学习软
|字数总计:4.4k|阅读时长:18分钟|阅读量:

实验内容

  1. 学习使用图形用户界面（GUI）设计实现一个简单的计算器，能进行+- ×/四则运算（注意数字大小的越界考虑，一般计算器输入的数字个数不超过16个）；
  2. 学习制作jar包，然后将jar包制作为exe可执行文件（可采用exe4j软件）；
  3. 实验报告粘贴Java代码，并截图运行效果，同时上传jar包，如果实现了exe可执行文件，同时上传exe文件。

code

CalculatorSelf.java

大概实现思路

该程序员计算器是在满足基础运算的情况下做了一些方便程序设计的拓展，拓展思路参考自win10程序员计算器：

但是其中一些功能并未实现：

  1. 不能在除十进制外的进制输入；
  2. 没有括号功能；
  3. 不能通过键盘输入（其实这个知道怎么实现，就是使用KeyListener，制作初期也想到要弄这个功能，但是打包完之后才想起来，想想貌似键盘输入的必要也不是很大）
  4. 在键入运算符后，不能更改运算符（但在这里采取了一个措施是按一次运算符则作为要进行一次运算）

其中第1行文本框为保存输入，第2行为输入数字，第3行结果的二进制转换，第4行为结果的八进制转换，第5行为结果的十六进制转换。

整体运行思路

通过按钮输入数字会显示在第2行，在按下一个输入运算符的按钮后，会保存第2行当前的数值，再判断前一次运算符，从而将当前输入框的数值进行运算，更新结果（正因如此，所以在按下一个运算符后无法更改），结果会显示在第2行，而二进制、八进制、十六进制框会随着第2行数值的更新而更新。

左移和右移的策略是对二进制进行一位的左/右移操作。

优化

优化主要是对界面的优化

  1. 按钮和文本框会随着Jframe的大小而变化；
  2. 按钮做了取消聚焦、鼠标移动到按钮上就会出现颜色变化（具体颜色也是取色自win10计算器）的优化；
  3. 按下一次“=”号后，再次输入会刷新第1行的显示。
